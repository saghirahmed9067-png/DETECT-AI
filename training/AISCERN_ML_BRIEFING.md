PROJECT OVERVIEW
───────────────
Aiscern (aiscern.com) is a multi-modal AI content detection SaaS platform.
It detects whether content (text, images, audio, video) was generated by AI
or created by a human. The platform serves real users and needs production-
grade fine-tuned models to replace the current generic pre-trained baselines.

PLATFORM STACK
──────────────
Frontend:   Next.js 14, deployed on Vercel
Inference:  HuggingFace Inference API (called from Vercel serverless functions)
Dataset:    saghi776/detectai-dataset (HuggingFace Hub, JSONL, ~413k rows)
Model Hub:  All fine-tuned models must be pushed to HuggingFace under
            the namespace: saghi776/
Auth:       Clerk
Database:   Supabase (PostgreSQL) for scan history, user profiles

TOOL 1: TEXT DETECTOR
═══════════════════════════════════════════════════════════════════════════

CURRENT ARCHITECTURE
────────────────────
3-model ensemble + 7 deterministic linguistic signals

ML Models (called via HF Inference API):
  Model A: openai-community/roberta-base-openai-detector  (weight: 0.40)
  Model B: Hello-SimpleAI/chatgpt-detector-roberta        (weight: 0.35)
  Model C: PirateXX/AI-Content-Detector                   (weight: 0.25)

Adaptive weighting: if a model returns error, its weight redistributes
to surviving models. ML ensemble accounts for 70% of final score.

7 Deterministic Linguistic Signals (30% of final score):
  1. Perplexity Proxy       — bigram repetition rate (AI text is "too smooth")
  2. Burstiness Score       — coefficient of variation of word spacing
                              (human writing has naturally bursty rare words)
  3. Type-Token Ratio       — MATTR vocabulary richness over 50-word windows
                              (AI score: 0.60–0.78 MATTR range)
  4. Zipf Deviation         — word frequency deviation from power law
  5. Sentence Uniformity    — CV of sentence lengths (AI < 0.20 CV → 0.82 score)
  6. AI Phrase Fingerprint  — 60+ LLM-overused phrases: "additionally",
                              "furthermore", "it's worth noting", "delve",
                              "comprehensive", "multifaceted", etc.
  7. Punctuation Entropy    — Shannon entropy of punctuation patterns
                              (AI punctuation is systematic, not idiosyncratic)

VERDICT THRESHOLDS
  AI:        final_score >= 0.62
  HUMAN:     final_score <= 0.38
  UNCERTAIN: 0.38 < score < 0.62

SENTENCE-LEVEL OUTPUT
  Each scan returns per-sentence AI scores + perplexity estimates for
  a heatmap visualization in the UI (up to 20 sentences shown).

INPUT CONSTRAINTS
  Max input: 512 tokens (truncated before sending to HF models)
  Min input: 50 characters (rejected below this threshold)
  Accepted: plain text, copy-pasted content, PDF text extraction

TRAINING DATA AVAILABLE NOW
  saghi776/detectai-dataset, config: text_en
  ~267,000 rows already on HF + ~146,000 in pipeline
  Fields: text, label (ai/human), quality (0-1), source, language
  Sources: HC3, RAID, Ghostbuster, Dolly-15k, Alpaca, OpenHermes,
           Wikipedia, CNN-DailyMail, arXiv, PubMedQA, StackExchange,
           OpenWebText, IMDB, Yelp, HH-RLHF, AG-News, Reddit-ELI5

TARGET MODEL TO FINE-TUNE
  Base:      roberta-base (HuggingFace)
  Task:      SequenceClassification, num_labels=2 (0=human, 1=ai)
  Output to: saghi776/aiscern-text-detector
  Target:    F1 >= 0.92, Accuracy >= 0.91

TRAINING HYPERPARAMETERS (recommended)
  epochs:              3
  learning_rate:       2e-5
  batch_size:          16 (fp16 enabled)
  warmup_steps:        200
  weight_decay:        0.01
  max_length:          512
  evaluation_strategy: epoch
  metric:              F1

INTEGRATION POINT IN CODEBASE
  File: frontend/lib/inference/hf-analyze.ts
  Line: MODELS.text_primary = 'openai-community/roberta-base-openai-detector'
  After fine-tuning: change to 'saghi776/aiscern-text-detector'
  The ensemble weights and signal logic remain unchanged.

TOOL 2: IMAGE DETECTOR (DEEPFAKE / AI-GENERATED)
═══════════════════════════════════════════════════════════════════════════

CURRENT ARCHITECTURE
────────────────────
3 HF vision models + 10 deterministic pixel-level signals + live calibration

ML Models:
  Model A: umm-maybe/AI-image-detector    (weight: 0.40) — general AI detector
  Model B: Organika/sdxl-detector         (weight: 0.35) — SDXL specialist
  Model C: Nahrawy/AIorNot               (weight: 0.25) — broad AI/real classifier

Adaptive ensemble: ML 65%, pixel signals 35%
(if all ML fail: pixel signals carry 100%)

10 Deterministic Pixel Signals:
  1. Byte Entropy          — AI images compress cleaner (lower entropy)
                             Gemini/Imagen < 6.2 bits → 0.90 score
                             SDXL/MJ 6.2-6.8 → 0.80 score
                             Real photos > 7.5 → 0.18 score
  2. DCT Coefficient       — frequency domain anomalies in JPEG blocks
  3. Noise Pattern         — sensor noise vs GAN smoothness analysis
  4. Color Distribution    — AI images have unnaturally uniform color histograms
  5. Edge Sharpness        — AI generators produce inconsistent edge profiles
  6. Compression Artifact  — JPEG quantization table analysis
  7. Metadata Presence     — AI generators strip/modify EXIF data
  8. Aspect Ratio          — AI generators favor specific ratios
  9. File Size Ratio       — AI images have unusual size/dimension ratios
  10. Edit Signature       — Photoshop/Lightroom editing artifacts

LIVE CALIBRATION
  Calibration stats loaded dynamically from Supabase at inference time.
  Recalibrates pixel signal thresholds using DiffusionDB ground truth samples.
  Falls back to static thresholds if calibration unavailable.

VERDICT THRESHOLDS (more aggressive than text)
  AI:        final_score >= 0.50 (lower threshold — better recall)
  HUMAN:     final_score <= 0.36
  UNCERTAIN: 0.36 < score < 0.50

AI GENERATORS EXPLICITLY TARGETED
  Stable Diffusion / SDXL / Flux
  Midjourney v5/v6
  DALL-E 3 (ChatGPT image generation)
  Google Gemini / Imagen / Gemini Nano
  Grok (Aurora model)
  Leonardo AI
  Adobe Firefly / Canva AI
  Runway, Pika, Kling (video frame extraction)

INPUT CONSTRAINTS
  Max size:  10MB
  Formats:   JPEG, PNG, WebP, HEIC
  Buffer:    Raw binary passed directly to HF API (application/octet-stream)

TRAINING DATA AVAILABLE
  saghi776/detectai-dataset, config: image_en
  Currently: ~745 rows (CRITICAL — not enough to fine-tune)
  Need: minimum 20,000 images before fine-tuning is viable
  Sources in pipeline: DiffusionDB, Midjourney-v6, CiFAKE,
                       Deepfake-Faces, AI-Art-LAION, DALLE3-COCO,
                       Unsplash-25k, Flickr30k, CelebA-HQ

TARGET MODEL TO FINE-TUNE
  Base:      google/vit-base-patch16-224 or microsoft/swin-base-patch4-window7-224
  Task:      ImageClassification, num_labels=2
  Output to: saghi776/aiscern-image-detector
  Target:    F1 >= 0.90

NOTE: Cannot fine-tune yet. Need 4-6 more weeks of image data collection.
Use existing models until dataset reaches 20k+ images.

INTEGRATION POINT

  Model B: HyperMoon/wav2vec2-base-960h-finetuned-deepfake (weight: 0.50)

  4. Byte Entropy           — TTS audio is more compressible (lower entropy)

  5. Spectral Pattern       — Header byte analysis for TTS spectral signatures

TOOL 4: VIDEO DETECTOR (DEEPFAKE VIDEO)
═══════════════════════════════════════════════════════════════════════════

CURRENT ARCHITECTURE
────────────────────
NVIDIA NIM API (primary) + 5 temporal/visual signals + heuristic fallback

Primary: NVIDIA NIM Vision API (meta/llama-3.2-90b-vision-instruct or
                                meta/llama-3.2-11b-vision-instruct fallback)
  — Analyzes up to 24 extracted frames per video
  — Frame extraction done in browser via HTMLVideoElement + canvas
  — Frames sent as base64 JPEG to NVIDIA API

5 Video Signals:
  1. NIM Mean Score         — average AI score across all analyzed frames (weight: 0.40)
  2. Face Region Score      — AI score on face-containing frames specifically (weight: 0.25)
                              (deepfakes must render faces; face frames are key signal)
  3. Worst Frame Score      — maximum suspicion across single frame (weight: 0.15)
  4. Temporal Inconsistency — high variance between frames = deepfake artifacts (weight: 0.12)
  5. Score Range Signal     — inconsistent frame-to-frame scores = manipulation (weight: 0.08)

FALLBACK (when NVIDIA NIM key missing or API fails)
  File-based heuristic: file size ratio + format signature
  Very low accuracy — this is the current state for most users

FRAME EXTRACTION
  Browser-side: user's browser extracts frames via canvas API
  Adaptive FPS: 1fps for short clips, up to 24fps sampling for longer
  Face detection: client-side MediaPipe face detection
  Frames sent to server as base64 JPEG array

VERDICT THRESHOLDS
  Same: AI >= 0.62, HUMAN <= 0.38

DEEPFAKE METHODS TARGETED
  FaceSwap / DeepFaceLab
  FaceForensics++ (all manipulation types)
  StyleGAN / StyleGAN2 / StyleGAN3
  DALL-E Video / Sora
  Runway Gen-2 / Gen-3
  Pika Labs
  Kling AI

FRAME-LEVEL OUTPUT
  Returns per-frame AI scores + face detection flags
  Up to 24 frame scores shown in UI grid

INPUT CONSTRAINTS
  Max size:  100MB
  Formats:   MP4, MOV, WebM, AVI
  Processing: frames extracted client-side, only base64 JPEGs sent to server

TRAINING DATA AVAILABLE
  saghi776/detectai-dataset, config: video_en
  Currently: 0 rows (CRITICAL — nothing collected yet)

TARGET MODEL TO FINE-TUNE

from datasets import load_dataset

  "id":           "uuid",
  "media_type":   "text" | "image" | "audio" | "video",
  "source":       "source-name",        # e.g. "hc3-english", "raid-benchmark"
  "source_dataset": "HF dataset id",    # e.g. "Hello-SimpleAI/HC3"
  "label":        "ai" | "human",       # binary classification target
  "quality":      0.0-1.0,              # quality score (filter >= 0.5)
  "text":         "full text content",  # for text modality
  "url":          "https://...",        # for image/audio/video
  "language":     "en",                 # ISO 639-1
  "split":        "train"|"val"|"test", # pre-split (90/5/5)
  "word_count":   312,
  "scraped_at":   "2026-03-19T..."
}

RECOMMENDED PREPROCESSING
  1. Filter: quality >= 0.5
  2. Filter: text length >= 100 characters (for text modality)
  3. Map labels: "ai" → 1, "human" → 0
  4. Use pre-existing split column for train/val/test
  5. For text: truncate to 512 tokens


═══════════════════════════════════════════════════════════════════════════
MODEL OUTPUT FORMAT (what the website expects from HF Inference API)
═══════════════════════════════════════════════════════════════════════════

After fine-tuning, the model must return HF standard classification format:

[
  [
    { "label": "LABEL_1", "score": 0.94 },  // AI probability
    { "label": "LABEL_0", "score": 0.06 }   // Human probability
  ]
]

OR with named labels:
[
  [
    { "label": "ai-generated", "score": 0.94 },
    { "label": "human-written", "score": 0.06 }
  ]
]

The website code at hf-analyze.ts:parseHFClassification() handles both
LABEL_0/LABEL_1 format and named label format automatically.

QUESTIONS THE DATA SCIENTIST MIGHT HAVE
═══════════════════════════════════════════════════════════════════════════

Q: What's the class balance in the text dataset?
A: 81,582 AI rows vs 64,882 human rows in current D1 batch.
   Roughly 55/45 AI/human. Apply class_weight='balanced' in training.

Q: Are there multiple languages?
A: 99%+ English (en). A small amount of multilingual text.
   Fine-tune on English only for now.

Q: What's the longest text in the dataset?
A: Capped at 4,000 characters (about 800-1000 tokens).
   Truncate all to 512 tokens for RoBERTa training.

Q: Is there a test set held out?
A: Yes — the 'split' column has train/val/test (90/5/5 split).
   Use split='test' rows for final evaluation only.

Q: Should we use the full ensemble or single model?
A: Fine-tune a single strong model first.
   The website ensemble handles multiple models automatically.
   A strong single model beats a weak ensemble.

Q: What evaluation metrics matter most?
A: F1 weighted (both classes matter equally).
   Precision on AI class (minimize false positives — bad UX).
   Recall on AI class (catch AI content — core product value).
   AUC-ROC for threshold tuning.

Q: Any class-specific requirements?
A: Prefer higher precision over recall on the AI class.
   A false positive (calling human content AI) is worse for UX
   than a false negative (missing AI content).
